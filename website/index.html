Machine Learning Models

Model 1: Baseline (Prophet)

Univariate Time Series

Configuration

  • Input: Historical step counts only
  • Seasonality: Yearly + Weekly
  • Train/Test: 80/20 split

Strengths

  • ✓ Simple and interpretable
  • ✓ Captures seasonal patterns
  • ✓ Robust to missing data

Limitations

  • ✗ Ignores clinical context
  • ✗ Cannot leverage features

Model 2: Advanced (EBM)

Explainable Boosting Machine

Configuration

  • Input: Step history + 40+ clinical features
  • Algorithm: Gradient boosting with GAMs
  • Interpretability: Built-in global explanations

Strengths

  • ✓ Incorporates clinical context
  • ✓ Fully explainable (glass-box)
  • ✓ Captures non-linear relationships
  • ✓ Feature importance analysis

Use Case

  • ✓ Healthcare decision support
  • ✓ Quantifies treatment impact

Frequently Asked Questions

What is the purpose of this project?

This project predicts daily step counts for patients using advanced ML models to help healthcare providers monitor recovery progress and adjust treatment plans proactively.

How accurate are the predictions?

The EBM model achieves an R² score of 0.87 with predictions typically within ±1,000-1,500 steps of actual values, including 95% confidence intervals.

What features are most important?

Top features include previous day's steps (28%), 7-day lag (22%), rolling averages (18-15%), day of week (12%), and clinical factors like active therapies and side effects.

How do I export results?

Use export buttons in the Data Explorer or keyboard shortcuts: Ctrl+E for CSV, Ctrl+P for PDF. Press ? to see all shortcuts.

Can I use this for my own data?

Yes! Prepare your data with daily step counts, timestamps, and clinical features, then retrain the models. The codebase is designed to be adaptable for similar healthcare forecasting tasks.
